bhatti

Firecracker microVM orchestrator

bhatti creates isolated Linux VMs in seconds. Each sandbox has its own kernel, filesystem, and network. Paused sandboxes resume in under 3ms.

bhatti completion

Generate shell completion script

bhatti create

Create a new sandbox

Create a new sandbox VM. Each sandbox is an isolated Linux environment with its own kernel, filesystem, and network.
  bhatti create --name dev
  bhatti create --name ml --cpus 4 --memory 4096
  bhatti create --name api --env API_KEY=sk-abc --init "npm install"
  bhatti create --name py --image python-3.12
  bhatti create --name work --volume workspace:/workspace
  bhatti create --name agent --init "hermes gateway" --keep-hot
FlagDefaultDescription
--cpus 1 Number of vCPUs
--disk-size 0 Rootfs disk size in MB (0 = use image size)
--env — Environment variables (K=V,K=V)
--image — Rootfs image name
--init — Init script
--keep-hot false Prevent thermal transitions (for autonomous agents)
--memory 0 Memory in MB (0 = server default: 2048)
--name — Sandbox name
--template — Template name or ID
--volume [] Persistent volume (name:mount[:ro])

bhatti destroy

aliases: rm

Destroy a sandbox

Permanently destroy a sandbox and all its data. Persistent volumes are detached but not deleted.
  bhatti destroy dev
FlagDefaultDescription
--yes, -y false Skip confirmation

bhatti edit

Update sandbox settings

Update mutable settings on an existing sandbox. Currently supports toggling keep_hot to control thermal management.
  bhatti edit my-agent --keep-hot
  bhatti edit my-agent --allow-cold
FlagDefaultDescription
--allow-cold false Re-enable thermal transitions
--keep-hot false Prevent thermal transitions

bhatti exec

Execute a command in a sandbox

Execute a command inside a sandbox. The exit code is forwarded. Sleeping sandboxes wake automatically.
  bhatti exec dev -- echo hello
  bhatti exec dev -- sudo apt-get install -y ripgrep
  bhatti exec dev --timeout 60 -- long-running-script.sh
FlagDefaultDescription
--timeout 0 Exec timeout in seconds (default: 300, max: 3600)

bhatti secret

Manage encrypted secrets

Secrets are encrypted at rest (age) and scoped to your API key. They can be referenced in templates and injected into sandboxes at boot.
  bhatti secret set API_KEY sk-abc123
  bhatti secret list
  bhatti secret delete API_KEY

bhatti shell

aliases: sh

Open an interactive shell

Open an interactive terminal inside the sandbox. Ctrl+\ to detach — the shell keeps running. Reconnect with 'bhatti shell' again.
  bhatti shell dev
FlagDefaultDescription
--new false Force a new session (don't reattach)

bhatti snapshot

Manage named VM snapshots

Snapshots capture the entire VM state: memory, CPU, disk. Resume produces an exact continuation — processes running, files open.
  bhatti snapshot create dev --name dev-ready
  bhatti snapshot resume dev-ready --name dev-2
  bhatti snapshot list

bhatti start

Resume a stopped sandbox

Resume a sandbox from its snapshot. Continues exactly where it left off.
  bhatti start dev

bhatti stop

Snapshot and stop a sandbox

Pause the sandbox and save a snapshot to disk. Resume later with 'bhatti start'. Stopped sandboxes use zero CPU and memory.
  bhatti stop dev

bhatti volume

Manage persistent volumes

Persistent volumes are ext4 filesystems that survive sandbox destruction. Attach them with '--volume name:/mount' on create.
  bhatti volume create --name workspace --size 5120
  bhatti create --name dev --volume workspace:/workspace
  bhatti volume resize workspace --size 10240
